# Sugar Diary（砂糖日记）

Sugar Diary 是一个以 **多 Agent 心理分析** 为核心的原生 Android 项目。它把日记正文、图片线索、历史情绪记录和用户画像组织成上下文，让多个专职 Agent 从不同心理视角并行分析、互相补充，最后由综合 Agent 生成结构化结果。

> 当前版本：2.0.0

## 项目亮点：多 Agent 心理分析系统

传统的“情绪分析”通常只给出一个标签或一段总结。Sugar Diary 的目标不是简单判断“开心/难过”，而是把一篇私人记录拆成多个可解释的心理观察维度：触发事件、自动想法、深层需求、关系线索、防御应对、身体压力、资源优势与安全风险。

应用内置 8 个专项心理 Agent：

| Agent | 关注点 |
| --- | --- |
| 触发情绪 Agent | 识别触发事件、情绪链路和即时心理波动 |
| 认知模式 Agent | 观察自动想法、解释风格和可能的认知偏差 |
| 需求价值 Agent | 提取未被满足的需求、价值冲突和意义感线索 |
| 关系依恋 Agent | 分析关系互动、边界张力、依恋与支持系统 |
| 防御应对 Agent | 识别回避、压抑、讨好、控制等应对模式及代价 |
| 身体压力 Agent | 关注睡眠、疲劳、紧绷、饮食和躯体压力信号 |
| 资源优势 Agent | 捕捉韧性、行动能力、边界尝试和可用支持 |
| 风险安全 Agent | 识别自伤、伤人、极端绝望或现实安全风险线索 |

这些 Agent 会先进行专项分析；在多 Agent 模式下，还会读取彼此观点，补充遗漏、修正过度推断；最后由综合 Agent 生成最终 JSON 结果，供 UI 展示、历史追踪、周期报告和后续对话使用。

## Agent 工作流

```text
日记正文 + 图片摘要 + 历史分析 + 用户画像
        │
        ▼
心理上下文构建器 PsychologyAgentContextBuilder
        │
        ▼
8 个专项 Agent 并行分析
        │
        ▼
跨 Agent 补充 / 反驳
        │
        ▼
综合 Agent 输出结构化 PsychologyAnalysisResult
        │
        ▼
分析详情、心理对话、周期报告、用户画像更新
```

核心实现位于：

- `domain/psychology/PsychologyAgentCatalog.kt`：定义 Agent 列表、职责和 prompt。
- `domain/psychology/PsychologyAgentOrchestrator.kt`：调度并行分析、跨 Agent 反驳和最终综合。
- `domain/psychology/PsychologyAgentContextBuilder.kt`：构建日记、历史分析和用户画像上下文。
- `domain/psychology/PsychologyJsonExtractor.kt`：修复和提取 LLM 返回的 JSON。
- `domain/psychology/UserPsychologyProfileMerger.kt`：把分析结果沉淀为长期用户画像。

## 可观察的 Agent 过程

Sugar Diary 不只保存最终结论，也保存 Agent 运行过程。用户可以在分析详情页看到：

- 哪个 Agent 正在工作。
- 每个 Agent 的专项分析摘要。
- 哪些观点被后续 Agent 补充或修正。
- 综合 Agent 如何把多个视角整合成最终结果。

这让心理分析不再是一个黑箱回复，而是更像一次由多个分析角色共同完成的可追踪推理过程。

## 心理 Agent 对话

完成单篇分析后，用户可以继续进入心理 Agent 对话。对话会结合：

- 当前日记内容。
- 最新分析结果。
- 历史分析摘要。
- 本地用户画像。

用户可以围绕“我为什么会这样反应”“下一步怎么照顾自己”“这是不是重复出现的模式”等问题继续追问。它不是医学诊断，而是一个面向自我理解的 Agentic reflection 工具。

## 用户画像与长期记忆

项目引入本地用户画像模型，用于记录长期线索：

- 长期触发点
- 认知倾向
- 核心需求
- 关系模式
- 防御/应对方式
- 身体压力信号
- 资源优势
- 风险注意事项

用户画像不是静态配置，而是可以由分析结果逐步更新，也可以由用户手动编辑。后续 Agent 分析会读取这些信息，从而形成“单篇分析 + 长期模式”的组合视角。

## 周期报告

在单篇分析之外，Sugar Diary 还能生成周报和月报。周期报告会汇总一段时间内的情绪标签、常见触发点、重复模式和建议，使 Agent 系统从“理解某一篇日记”扩展到“观察一段时间的心理趋势”。

## 普通日记能力

虽然项目重点是 Agent 心理分析，但应用仍提供完整的本地写作基础：

- Markdown / HTML 内容编辑与预览。
- 本地图片嵌入和缩略图预览。
- 时间轴、浏览、搜索、标签和心情筛选。
- AI 格式转换和文风润色。
- 完整迁移包和原始文章目录导入导出。

## 模型配置

应用使用 OpenAI 兼容 Chat Completions 接口。你需要配置：

- 主模型接口地址、模型名、API Key。
- 请求超时时间。
- 可选的独立图片理解模型。

多 Agent 分析会产生多次模型调用，建议使用响应稳定、上下文长度充足的模型，并适当放宽超时时间。

## 2.0.0 签名重置提醒

由于旧版 release 签名证书已无法找回，`2.0.0` 使用新的正式签名证书发布。Android 会把它视为不同签名的安装包，通常不能直接覆盖安装 `1.x` 版本。

从 `1.x` 升级前请先在旧版应用中导出完整迁移包或原始文章目录，然后卸载旧版，再安装 `2.0.0` 并导入数据。


## 重要声明

Sugar Diary 的心理分析功能用于写作辅助、自我观察和心理教育，不替代心理咨询、医学诊断或危机干预。如果你正在经历现实风险或强烈痛苦，请优先联系现实中的可信支持、当地紧急服务或专业人员。
